# Contributing

How to contribute

1. Fork the repository and create a feature branch: `git checkout -b my-feature`.
2. Make your changes on the feature branch. Keep changes focused and small.
3. Run the site locally to verify visual changes by opening `index.html` in a browser (no build step required). For a simple server, use:

```bash
python -m http.server 8000
```

4. Commit with a clear message and push to your fork.
5. Open a Pull Request against `main` with a short description of your change and any screenshots if applicable.

Guidelines

- Code style: prefer clear, readable JavaScript and CSS. Follow existing project conventions.
- Tests: this project has no automated tests; manual verification in the browser is sufficient.
- Accessibility: keep semantic HTML and provide alt text for images where appropriate.
- Large changes: open an issue first to discuss scope before implementing.

License and CLA

By contributing, you agree that your contributions will be licensed under the repository's existing license.

Contact

If you have questions, open an issue or reach out via the contact links in the site header/footer.
